# NB - Guide - Tagging

Tags: #nb #guide #tags #taxonomy

## Purpose
Instructions for future agents on how to add and choose nb tags consistently.

## When to Add Tags
- Use tags when a note centers on a reusable concept or workflow that someone might search later.
- Prefer tags for problems or mechanisms that recur (e.g., #cors, #hover, #layout, #zoom).
- Skip tags for one-off status updates unless they add search value.

## How to Choose Tags
- **Concept density**: pick the 2â€“5 most important concepts in the note.
- **Implementation surface**: tag for code areas or technologies that would help future lookup (e.g., #d3, #svg, #python).
- **User intent**: tag for UX/troubleshooting themes that motivated the change (e.g., #ux, #labels).
- **Avoid redundancy**: choose one canonical tag rather than multiple synonyms (e.g., #ux instead of #usability-ux and #usability).

## Where to Apply Tags
- Add broad tags to hubs or summaries; add focused tags to detail notes.
- For worklog details, tag by the dominant topic of the doc.

## Example
- A D3 layout note: `Tags: #nb-visual #d3 #layout #graph #svg`
- A CORS fix note: `Tags: #nb-visual #cors #html`
